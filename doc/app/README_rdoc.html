    Prerequisites¶ ↑

    • bundler

    • git

    • Node.js

    • PostgreSQL

    • Ruby 2.x

    Installation¶ ↑

    On Debian 8 (Jessie) or later and Ubuntu 14.10 (Utopic Unicorn) or later:

    # apt-get install bundler git nodejs postgresql postgresql-server-dev-9.4 ruby ruby-dev

    On Ubuntu 14.04 (Trusty Tahr):

    # apt-get install bundler git nodejs postgresql postgresql-server-dev-9.3 ruby2.0 ruby2.0-dev

    # rm /usr/bin/ruby /usr/bin/gem /usr/bin/irb /usr/bin/rdoc /usr/bin/erb

    # for f in ruby gem irb rdoc erb; do ln -s /usr/bin/${f}2.0 /usr/bin/$f; done

    # gem update --system

    # gem pristine --all

    Then, on all systems:

    $ git clone https://[username]@github.com/valegro/subsright.git

    $ cd subsright

    $ git checkout master

    $ bundle install

    $ sudo gem pristine --all

    Development¶ ↑

    $ sudo -u postgres createuser -d $USER

    $ rake db:setup

    $ rake spec

    $ STRIPE_PUBLISHABLE_KEY='test pk' STRIPE_SECRET_KEY='test sk' rails s

    You can then log at localhost:3000/admin as admin@example.com with the initial default password “password”. You can now create and edit admin users and the site configuration.

    You can then visit localhost:3000 to see the public interface.

    Production¶ ↑

    $ sudo -u postgres createuser -d -P subsright

    $ SUBSRIGHT_DATABASE_PASSWORD='password' RAILS_ENV=production rake db:setup

    $ SUBSRIGHT_DATABASE_PASSWORD='password' RAILS_ENV=production rake assets:precompile

    $ rake secret

    <tt>$ STRIPE_PUBLISHABLE_KEY='live pk' STRIPE_SECRET_KEY='live sk' SECRET_KEY_BASE='secret'

    SUBSRIGHT_DATABASE_PASSWORD='<em>password</em>' RAILS_ENV=production bundle exec rackup -E production</tt>

    You can then log at localhost:9292/admin as admin@example.com with the initial default password “password”. You can now create and edit admin users and the site configuration.

    You can then visit localhost:9292 to see the public interface.
